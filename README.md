# VK Album Downloader

Простой скрипт для массового скачивания фотографий из **публичных** альбомов ВКонтакте в максимальном качестве.

## Зачем это нужно?

Если вам нужно сохранить фотографии из альбома ВК (например, с мероприятия, свадьбы или корпоратива), этот скрипт сделает всё за вас. Он умеет:

- Скачивать фото в максимально доступном качестве
- Работать с большими альбомами (сотни и тысячи фотографий)
- Качать несколько фото одновременно (многопоточность)
- Пропускать уже скачанные файлы
- Находить и не скачивать дубликаты
- Обрабатывать целый список альбомов из CSV-файла

**⚠️ Важно:** Скрипт работает только с **публичными** альбомами ВКонтакте. Закрытые альбомы и альбомы, требующие авторизации, недоступны для скачивания.

### Работает без токена

Скрипт скачивает публичные альбомы **без какой-либо авторизации** — просто запустите его. Внутри код разбит на пакет `vkdl/` (`config`, `quality`, `scraper`, `downloader`, `api_client`, `resolver`, `batch`), а `vk_downloader.py` — тонкий CLI поверх него.

**Опционально:** если задать переменную окружения `VK_ACCESS_TOKEN`, скрипт сначала попробует официальный VK API (надёжнее при редизайнах сайта), а при любой ошибке автоматически вернётся к скачиванию через парсинг. Токен не обязателен.

```bash
export VK_ACCESS_TOKEN=...   # необязательно
```

## Установка и запуск

### Вариант 1: С помощью uv (рекомендуется)

Если у вас установлен [uv](https://github.com/astral-sh/uv), просто запускайте скрипт напрямую — он сам установит все зависимости:

```bash
uv run vk_downloader.py https://vk.com/album-18515186_240802273
```

### Вариант 2: Через pip и виртуальное окружение

Если предпочитаете классический подход:

```bash
# Создаём виртуальное окружение
python3 -m venv venv

# Активируем его
source venv/bin/activate  # для Linux/macOS
# или
venv\Scripts\activate     # для Windows

# Устанавливаем зависимости
pip install requests beautifulsoup4

# Запускаем скрипт
python vk_downloader.py https://vk.com/album-18515186_240802273
```

## Параметры запуска

### Одиночный режим

Скачивает один альбом:

```bash
uv run vk_downloader.py <URL_альбома> [количество_потоков]
```

**Параметры:**
- `URL_альбома` — ссылка на альбом ВК (обязательный)
- `количество_потоков` — сколько фото качать одновременно (по умолчанию 5, максимум 20)

### Пакетный режим

Обрабатывает сразу несколько альбомов из CSV-файла:

```bash
uv run vk_downloader.py --batch <CSV_файл> [количество_потоков]
```

**Параметры:**
- `--batch` — флаг пакетного режима
- `CSV_файл` — путь к файлу со списком альбомов (обязательный)
- `количество_потоков` — сколько фото качать одновременно (по умолчанию 5, максимум 20)

**Формат CSV-файла:**

```csv
Name;DateStart;AlbumLink
Корпоратив 2024;2024-12-25 18:00:00;https://vk.com/album-123456_789012
Летний пикник;2024-07-15 12:00:00;https://vk.com/album-123456_789013
```

Можно использовать как точку с запятой (`;`), так и запятую (`,`) в качестве разделителя.

## Примеры использования

### Скачать один альбом с настройками по умолчанию:
```bash
uv run vk_downloader.py https://vk.com/album-18515186_240802273
```

### Скачать альбом с 10 потоками (быстрее):
```bash
uv run vk_downloader.py https://vk.com/album-18515186_240802273 10
```

### Обработать список альбомов из файла:
```bash
uv run vk_downloader.py --batch albums.csv
```

### Обработать список с повышенной скоростью:
```bash
uv run vk_downloader.py --batch albums.csv 15
```

## FAQ

### Скрипт пишет "Фотографии не найдены" или "Доступ запрещён"

**Проблема:** Альбом закрыт настройками приватности или требует авторизации.

**Решение:** К сожалению, скрипт работает только с публичными альбомами. Попросите владельца альбома открыть доступ или используйте браузерные расширения для скачивания (работая из авторизованного аккаунта).

### Скачиваются не все фото или качество хуже ожидаемого

**Проблема:** ВКонтакте может ограничивать доступ к оригиналам фотографий.

**Решение:** Скрипт автоматически пробует несколько вариантов качества, начиная с максимального. Если оригинал недоступен, он скачает лучшее из доступных. В логах будет пометка "(качество 2/3)" и т.п.

### Ошибка "ModuleNotFoundError: No module named 'requests'"

**Проблема:** Не установлены зависимости.

**Решение:** 
- Если используете `uv`, убедитесь, что запускаете через `uv run`
- Если используете pip, активируйте виртуальное окружение и установите зависимости: `pip install requests beautifulsoup4`

### Скрипт зависает или работает слишком медленно

**Проблема:** Возможны проблемы с сетью или слишком мало потоков.

**Решение:** 
- Попробуйте увеличить количество потоков до 10-15
- Проверьте скорость интернета
- Если альбом очень большой (1000+ фото), это нормально — просто подождите

### CSV файл не читается или выдаёт ошибки

**Проблема:** Неправильная кодировка или формат файла.

**Решение:**
- Сохраните CSV в кодировке UTF-8
- Проверьте, что есть заголовок: `Name;DateStart;AlbumLink`
- Убедитесь, что все три колонки заполнены в каждой строке
- Даты должны быть в формате `YYYY-MM-DD` или `YYYY-MM-DD HH:MM:SS`

### Скрипт скачал дубликаты

**Проблема:** На самом деле нет — скрипт умный!

**Решение:** Если видите сообщение "🔗 дубликат ...", это значит, что скрипт *не* скачал файл, потому что точно такая же фотография уже есть. Он сравнивает файлы по содержимому (SHA256), а не по имени.

### Можно ли прервать скачивание и продолжить потом?

**Да!** Просто нажмите `Ctrl+C` для остановки. При повторном запуске скрипт пропустит уже скачанные файлы (увидите "⏭️ пропущено").

### Скрипт ругается на количество потоков

**Проблема:** Указано слишком большое или маленькое число.

**Решение:** Используйте значение от 1 до 20. Оптимально 5-10 для большинства случаев.

### Фотографии скачиваются в странную папку

Скрипт создаёт папку с названием альбома (или датой+названием в пакетном режиме) в текущей директории. Чтобы изменить место сохранения, просто перейдите в нужную папку перед запуском:

```bash
cd ~/Downloads
uv run /путь/к/vk_downloader.py https://vk.com/album...
```

---

**Совет:** Начните с небольшого альбома и 5 потоков, чтобы убедиться, что всё работает. Потом можно увеличить скорость для больших задач.
